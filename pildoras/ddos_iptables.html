Protección contra DDoS con IPTables

Guía completa para configurar reglas de iptables y mitigar ataques de denegación de servicio

Introducción

Los ataques DDoS (Distributed Denial of Service) son una de las amenazas más comunes en la actualidad. IPTables, el firewall integrado en Linux, ofrece diversas técnicas para mitigar estos ataques.

Esta guía te proporcionará las reglas y configuraciones necesarias para proteger tu servidor contra diferentes tipos de ataques DDoS.

Configuración del Kernel

Antes de aplicar reglas de iptables, es crucial optimizar el kernel para mejorar el rendimiento bajo ataque DDoS.

Añade estas líneas en /etc/sysctl.conf:

# Ajustes clave para mitigar DDoS
net.ipv4.tcp_syncookies = 1
net.ipv4.tcp_max_syn_backlog = 16384
net.ipv4.tcp_synack_retries = 1
net.ipv4.tcp_syn_retries = 2
net.ipv4.icmp_echo_ignore_broadcasts = 1
net.ipv4.icmp_ignore_bogus_error_responses = 1
net.ipv4.tcp_rfc1337 = 1
net.core.netdev_max_backlog = 262144
net.core.somaxconn = 65535
net.netfilter.nf_conntrack_max = 10000000

Ejecuta sysctl -p para aplicar los cambios.

Reglas Básicas de IPTables

Estas reglas proporcionan una protección básica contra ataques DDoS:

# Bloquear paquetes inválidos
iptables -t mangle -A PREROUTING -m conntrack --ctstate INVALID -j DROP

# Bloquear nuevos paquetes que no son SYN
iptables -t mangle -A PREROUTING -p tcp ! --syn -m conntrack --ctstate NEW -j DROP

# Bloquear valores MSS anómalos
iptables -t mangle -A PREROUTING -p tcp -m conntrack --ctstate NEW -m tcpmss ! --mss 536:65535 -j DROP

Reglas Avanzadas de Protección

Estas reglas ofrecen protección contra tipos específicos de ataques:

Protección contra SYN Flood

# Limitar conexiones SYN por segundo
iptables -A INPUT -p tcp --syn -m limit --limit 1/s --limit-burst 3 -j ACCEPT
iptables -A INPUT -p tcp --syn -j DROP

Protección contra HTTP Flood

# Limitar conexiones HTTP
iptables -A INPUT -p tcp --dport 80 -m limit --limit 25/minute --limit-burst 100 -j ACCEPT
iptables -A INPUT -p tcp --dport 80 -j DROP

Protección contra ICMP Flood (Ping of Death)

# Crear cadena personalizada para ICMP
iptables -N ICMP_CHECK
iptables -A ICMP_CHECK -p icmp --icmp-type echo-request -m limit --limit 1/s --limit-burst 4 -j RETURN
iptables -A ICMP_CHECK -p icmp --icmp-type echo-request -m length --length 0:100 -j RETURN
iptables -A ICMP_CHECK -j DROP
iptables -A INPUT -p icmp --icmp-type echo-request -j ICMP_CHECK

Script de Bloqueo Automático

Crea un script que detecte y bloquee automáticamente IPs maliciosas:

#!/bin/bash
# dropip.sh - Auto-detectar y bloquear IPs sospechosas
netstat -na | awk '/ESTABLISHED/{split($5,T,":");print T[1]}' | sort | grep -v -E '192.168|127.0' | uniq -c | sort -rn | head -10 | awk '{if ($2!=null && $1>4) {print $2}}' > /var/log/rejectip

for i in $(cat /var/log/rejectip)
do
    rep=$(iptables-save | grep $i)
    if [[ -z $rep ]];then
        /sbin/iptables -A INPUT -s $i -j DROP
        echo "$i bloqueada en $(date)" >> /var/log/ddos-ip
    fi
done

Puedes programar este script para que se ejecute periódicamente con cron.

Conservar las Reglas

Para que las reglas persistan después de reiniciar:

# Instalar iptables-persistent
sudo apt-get install iptables-persistent

# Guardar reglas actuales
sudo iptables-save > /etc/iptables/rules.v4
sudo ip6tables-save > /etc/iptables/rules.v6

Recomendaciones Adicionales

  • Usa herramientas de monitorización como netstat, tcpdump, o vnstat
  • Para ataques masivos, considera servicios especializados como Cloudflare
  • Combina iptables con un WAF para proteger aplicaciones web
  • Mantén el sistema y iptables actualizados
  • Realiza pruebas regularmente para verificar la efectividad

⚠️ Advertencia Importante

Estas reglas deben adaptarse a las necesidades específicas de tu servidor y tráfico. Aplicar configuraciones demasiado restrictivas podría bloquear tráfico legítimo.

Ninguna solución es 100% efectiva contra ataques DDoS grandes, pero iptables ofrece una capa de defensa sólida y flexible.
