AWS GuardDuty

AWS GuardDuty es un servicio de detección de amenazas inteligente que utiliza machine learning, análisis de anomalías y fuentes de inteligencia de amenazas para identificar automáticamente amenazas activas y comportamientos potencialmente no autorizados o maliciosos en entornos AWS.

Características Principales

  • Detección inteligente de amenazas: Utiliza machine learning para identificar comportamientos anómalos
  • Análisis continuo: Monitorea continuamente VPC Flow Logs, CloudTrail y DNS logs
  • Inteligencia de amenazas integrada: Acceso a bases de datos globales de amenazas
  • Alertas priorizadas: Clasifica hallazgos por severidad y contexto
  • Integración nativa: Funciona automáticamente con servicios AWS existentes

Tipos de Amenazas Detectadas

  • Compromiso de credenciales: Uso no autorizado de credenciales IAM
  • Actividad de reconocimiento: Escaneo de puertos y enumeración de servicios
  • Instancias comprometidas: Servidores infectados con malware
  • Acceso no autorizado: Intentos de acceso a recursos protegidos
  • Exfiltración de datos: Transferencias inusuales de datos

Fuentes de Datos Analizadas

  • VPC Flow Logs: Análisis de tráfico de red dentro de VPCs
  • CloudTrail Events: Registro de actividad de API de AWS
  • DNS Logs: Consultas DNS realizadas por instancias EC2
  • EBS Volume Data: Análisis de volúmenes EBS adjuntos
  • Runtime Monitoring: Detección de amenazas en tiempo de ejecución
Importancia en Wazuh: GuardDuty puede integrarse con Wazuh para enriquecer las capacidades de detección de amenazas en entornos AWS, proporcionando una capa adicional de protección inteligente que complementa las funcionalidades nativas de Wazuh para análisis de logs y detección de anomalías.
