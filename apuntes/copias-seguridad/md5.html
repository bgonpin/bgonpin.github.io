MD5 - Algoritmo de Hash

Función hash criptográfica para verificación de integridad

Tipos de Backup Regla 3-2-1 RTO y RPO Software de Backup Verificación de Integridad

¿Qué es MD5?

Message Digest 5

MD5 es una función hash criptográfica que produce un valor hash de 128 bits (32 caracteres hexadecimales). Fue diseñado por Ronald Rivest en 1991 y se utiliza ampliamente para verificar la integridad de archivos y datos.

Características principales

  • Longitud del hash: 128 bits (32 caracteres hex)
  • Velocidad: Muy rápido en computación
  • Uso común: Verificación de descargas, integridad de archivos
  • Estado actual: Obsoleto para seguridad criptográfica

Cómo funciona MD5

Proceso de hash

MD5 procesa el mensaje de entrada en bloques de 512 bits, aplicando una serie de operaciones matemáticas para producir el hash final.

Pasos principales

  1. Preprocesamiento: Añade padding y longitud del mensaje
  2. Procesamiento en bloques: Divide el mensaje en bloques de 512 bits
  3. Compresión: Aplica función de compresión 64 veces por bloque
  4. Resultado: Produce hash de 128 bits

Uso en Verificación de Integridad

Aplicaciones comunes

  • Verificación de descargas: Comprobar integridad de archivos descargados
  • Control de cambios: Detectar modificaciones en archivos
  • Almacenamiento de contraseñas: (Obsoleto, no recomendado)
  • Verificación de backups: Asegurar que los datos no han cambiado

Ejemplo de uso

# Linux
md5sum archivo.txt

# Windows PowerShell
Get-FileHash -Algorithm MD5 archivo.txt

Seguridad y Limitaciones

Vulnerabilidades conocidas

  • Colisiones: Encontradas en 2004 por Xiaoyun Wang
  • Ataques de fuerza bruta: Fácil de computar en hardware moderno
  • Rainbow tables: Precomputación de hashes

¿Por qué está obsoleto?

  • No resiste ataques de colisión
  • Fácil de falsificar
  • Reemplazado por SHA-256 y superiores

Alternativas Recomendadas

Algoritmos más seguros

  • SHA-256: Estándar actual para integridad
  • SHA-3: Nuevo estándar NIST
  • BLAKE2: Alternativa moderna y rápida

Migración desde MD5

Para aplicaciones existentes, se recomienda migrar a algoritmos más seguros. MD5 aún puede usarse para verificación de integridad no criptográfica donde la velocidad es prioritaria sobre la seguridad.

Herramientas que usan MD5

Herramientas nativas

  • Linux: md5sum
  • Windows: CertUtil, PowerShell
  • macOS: md5

Aplicaciones

  • Gestores de descargas (verificación de integridad)
  • Herramientas de backup (Duplicati, etc.)
  • Software de verificación de archivos
