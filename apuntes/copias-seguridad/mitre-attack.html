MITRE ATT&CK Framework

Fecha de actualización: 26-08-2025

Resumen: El marco MITRE ATT&CK es un modelo global de conocimiento sobre tácticas y técnicas utilizadas por actores maliciosos durante un ciberataque. Desarrollado por MITRE Corporation, este framework proporciona una estructura común para describir el comportamiento de los adversarios y sirve como base para la detección, respuesta y mitigación de amenazas cibernéticas.

1. Introducción

MITRE ATT&CK (Adversarial Tactics, Techniques, and Common Knowledge) es un proyecto de código abierto que documenta tácticas, técnicas y procedimientos (TTPs) utilizados por actores maliciosos en entornos empresariales. Lanzado inicialmente en 2013, el framework ha evolucionado para cubrir múltiples plataformas incluyendo:

  • Enterprise: Windows, Linux, macOS, Network, Containers, Cloud
  • Mobile: Android, iOS
  • Industrial Control Systems (ICS)

2. Estructura del Framework

MITRE ATT&CK se organiza en tres niveles principales:

2.1 Táctica (Tactic)

Las tácticas representan el "por qué" o el objetivo estratégico del atacante. Cada táctica corresponde a una fase del ciclo de vida del ataque.

Principales Tácticas en MITRE ATT&CK Enterprise

2.2 Técnica (Technique)

Las técnicas describen "cómo" se lleva a cabo una táctica. Cada técnica tiene un identificador único (ej. T1197, T1562.002).

T1197 - BITS Jobs

Descripción: Uso del servicio Background Intelligent Transfer Service (BITS) para descargar y ejecutar código malicioso en segundo plano.

Táctica: Command and Control

T1562.002 - Disable Windows Event Logging

Descripción: Desactivación del registro de eventos de Windows para evitar la detección.

Táctica: Defense Evasion

2.3 Procedimiento (Procedure)

Los procedimientos son implementaciones específicas de técnicas observadas en la práctica. Representan el "qué" específico que hace el atacante.

3. Matrices de ATT&CK

3.1 Enterprise Matrix

La matriz Enterprise es la más completa y cubre 14 tácticas principales:

  1. Reconnaissance (Reconocimiento) - Recopilación de información sobre el objetivo
  2. Resource Development (Desarrollo de Recursos) - Establecimiento de infraestructura
  3. Initial Access (Acceso Inicial) - Entrada inicial al sistema
  4. Execution (Ejecución) - Ejecución de código malicioso
  5. Persistence (Persistencia) - Mantenimiento del acceso
  6. Privilege Escalation (Escalada de Privilegios) - Obtención de permisos superiores
  7. Defense Evasion (Evasión de Defensas) - Evasión de controles de seguridad
  8. Credential Access (Acceso a Credenciales) - Robo de credenciales
  9. Discovery (Descubrimiento) - Exploración del entorno
  10. Lateral Movement (Movimiento Lateral) - Navegación dentro de la red
  11. Collection (Recolección) - Recopilación de información valiosa
  12. Command and Control (Comando y Control) - Comunicación con sistemas comprometidos
  13. Exfiltration (Exfiltración) - Extracción de datos
  14. Impact (Impacto) - Manipulación, interrupción o destrucción de sistemas

3.2 Mobile Matrix

Específica para dispositivos móviles con tácticas adaptadas:

  • Initial Access
  • Execution
  • Persistence
  • Credential Access
  • Discovery
  • Lateral Movement
  • Collection
  • Command and Control
  • Exfiltration
  • Impact

4. Aplicaciones del Framework

4.1 Detección y Respuesta a Incidentes

MITRE ATT&CK se utiliza para:

  • Mapeo de alertas a tácticas específicas
  • Identificación de brechas en la cobertura de seguridad
  • Priorización de amenazas basadas en tácticas observadas
  • Desarrollo de reglas de detección para SIEM y EDR
Ejemplo: Una alerta de ejecución de PowerShell sospechoso puede mapearse a la táctica "Execution" con técnica T1059.001 (PowerShell).

4.2 Evaluación de Seguridad (Red Team/Blue Team)

Los equipos de seguridad utilizan ATT&CK para:

  • Simulación de ataques siguiendo tácticas reales
  • Evaluación de controles de seguridad
  • Mejora de la madurez del programa de seguridad
  • Entrenamiento y concienciación

4.3 Desarrollo de Productos de Seguridad

Los fabricantes utilizan ATT&CK para:

  • Mapeo de capacidades de sus productos
  • Validación de detección contra técnicas conocidas
  • Comunicación clara de capacidades a clientes
  • Investigación y desarrollo de nuevas características

5. Implementación Práctica

5.1 Mapeo de Detecciones

Ejemplo de regla de correlación para SIEM:


  60101
  (IEX|Invoke-Expression|DownloadString)
  Suspicious PowerShell execution
  T1059.001

5.2 Evaluación de Cobertura

Proceso para evaluar la efectividad de controles:

  1. Identificar tácticas críticas para la organización
  2. Mapear controles existentes a técnicas ATT&CK
  3. Identificar brechas en la cobertura
  4. Priorizar implementaciones basadas en riesgo

6. Herramientas y Recursos

6.1 Plataformas Oficiales

  • MITRE ATT&CK Navigator: Herramienta web para visualización y análisis
  • ATT&CK Workbench: Aplicación de escritorio para gestión local
  • ATT&CK for Enterprise/Mobile: Documentación completa

6.2 Integraciones Comunes

  • Splunk: Dashboards y alertas basadas en ATT&CK
  • IBM QRadar: Mapeo automático de eventos
  • CrowdStrike Falcon: Detecciones alineadas con ATT&CK
  • Microsoft Defender: Integración nativa

7. Mejores Prácticas

Recomendaciones para implementar MITRE ATT&CK en la organización:

7.1 Comenzar con lo Básico

  • Enfocarse en tácticas de alto impacto: Initial Access, Privilege Escalation, Lateral Movement
  • Mapear controles existentes antes de implementar nuevos
  • Utilizar el framework como lenguaje común entre equipos

7.2 Medición de Éxito

  • Porcentaje de técnicas con controles de detección
  • Tiempo medio de detección (MTTD) por táctica
  • Reducción en tiempo de respuesta a incidentes
  • Madurez en la identificación de amenazas avanzadas

7.3 Mantenimiento Continuo

  • Revisar actualizaciones del framework trimestralmente
  • Adaptar el mapeo a cambios en el entorno
  • Incorporar nuevas técnicas observadas
  • Entrenar al equipo en nuevas tácticas

8. Limitaciones y Consideraciones

8.1 Limitaciones del Framework

  • No es exhaustivo: No cubre todas las técnicas posibles
  • Enfoque en Enterprise: Menos aplicable a otros entornos
  • Sesgo hacia amenazas conocidas: Técnicas APT tienen mayor representación

8.2 Consideraciones de Implementación

  • Complejidad: El framework es extenso y requiere dedicación
  • Recursos necesarios: Requiere personal capacitado
  • Evolución constante: Requiere mantenimiento continuo

9. Conclusiones

MITRE ATT&CK representa un avance significativo en la comprensión y defensa contra amenazas cibernéticas. Al proporcionar un lenguaje común y una estructura organizada, facilita la comunicación entre equipos de seguridad, mejora la detección de amenazas y permite una respuesta más efectiva a incidentes.

La implementación exitosa requiere compromiso organizacional, capacitación continua y adaptación a las necesidades específicas de cada entorno. Cuando se utiliza correctamente, ATT&CK no solo mejora la seguridad técnica, sino que también fomenta una cultura de seguridad proactiva y consciente.
