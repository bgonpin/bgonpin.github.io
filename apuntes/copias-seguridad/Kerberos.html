¿Qué es Kerberos?

Kerberos es un protocolo de autenticación de red que utiliza criptografía de clave secreta para proporcionar autenticación fuerte en entornos de red no seguros. Fue desarrollado en el MIT (Massachusetts Institute of Technology) como parte del proyecto Athena y se ha convertido en un estándar de la industria para autenticación en redes.

Características principales

  • Autenticación mutua: Tanto el cliente como el servidor se autentican entre sí
  • Criptografía de clave secreta: Utiliza algoritmos simétricos para la encriptación
  • Tickets de autenticación: Sistema basado en tickets temporales
  • Single Sign-On (SSO): Una autenticación permite acceso a múltiples servicios
  • Seguridad robusta: Protege contra ataques de replay y spoofing

Componentes del sistema Kerberos

  • Centro de Distribución de Claves (KDC): Servidor central que emite tickets
  • Servidor de Autenticación (AS): Valida la identidad del usuario
  • Servidor de Concesión de Tickets (TGS): Emite tickets de servicio
  • Base de datos de principales: Almacena información de usuarios y servicios
  • Cliente Kerberos: Aplicación que solicita autenticación

Funcionamiento básico

  1. Autenticación inicial: El cliente solicita un TGT (Ticket Granting Ticket) al AS
  2. Obtención de ticket de servicio: El cliente usa el TGT para obtener un ticket de servicio del TGS
  3. Acceso al servicio: El cliente presenta el ticket de servicio al servidor
  4. Autenticación mutua: El servidor valida el ticket y responde con una confirmación

Usos comunes

  • Autenticación en dominios Windows (Active Directory)
  • Servicios de archivos compartidos (SMB, NFS con Kerberos)
  • Autenticación SSH con Kerberos
  • Servicios web con autenticación Kerberos (SPNEGO)
  • Integración con protocolos de red como AFP, SMB, y NFS

Implementaciones populares

  • Microsoft Active Directory: Implementación más utilizada en entornos Windows
  • MIT Kerberos: Implementación de referencia de código abierto
  • Red Hat Enterprise Linux: Integración nativa con Kerberos
  • Apple macOS: Soporte Kerberos para autenticación en red

Comandos básicos Kerberos

# Obtener ticket inicial (Linux)
kinit usuario@DOMINIO.COM

# Ver tickets actuales
klist

# Destruir todos los tickets
kdestroy

# Renovar ticket
kinit -R

# Configurar Kerberos (Windows)
ksetup /setdomain DOMINIO.COM
ksetup /addkdc DOMINIO.COM kdc.dominio.com

Configuración básica

# Archivo /etc/krb5.conf (Linux)
[libdefaults]
    default_realm = DOMINIO.COM
    dns_lookup_realm = false
    dns_lookup_kdc = false

[realms]
    DOMINIO.COM = {
        kdc = kdc.dominio.com
        admin_server = kdc.dominio.com
    }

Ventajas y limitaciones

Ventajas

  • Seguridad robusta contra ataques
  • Single Sign-On eficiente
  • Escalabilidad en grandes redes
  • Integración con múltiples servicios
  • Soporte en múltiples plataformas

Limitaciones

  • Complejidad de configuración
  • Sincronización de relojes requerida
  • Vulnerabilidades si se compromete el KDC
  • No es adecuado para autenticación web directa
  • Requiere infraestructura dedicada
