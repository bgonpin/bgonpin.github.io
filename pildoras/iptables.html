1. Introducción a iptables

iptables es el cortafuegos estándar de facto en sistemas Linux, proporcionando un framework robusto para el filtrado de paquetes, traducción de direcciones de red (NAT) y manipulación de paquetes. Forma parte del proyecto Netfilter del kernel de Linux y ha sido la herramienta principal para la seguridad de red en Linux durante más de dos décadas.

💡 Datos Interesantes:
  • iptables fue desarrollado por Rusty Russell en 1998
  • Reemplazó a ipchains e ipfwadm en versiones anteriores de Linux
  • Actualmente está siendo gradualmente reemplazado por nftables en distribuciones modernas

¿Por qué usar iptables?

iptables ofrece ventajas significativas para la administración de la seguridad de red:

  • Control granular: Permite definir reglas específicas para diferentes tipos de tráfico
  • Flexibilidad: Soporta múltiples criterios de filtrado y acciones
  • Rendimiento: Procesamiento a nivel de kernel para máxima eficiencia
  • Estabilidad: Probado en entornos de producción durante años

2. Conceptos Fundamentales

Tablas

iptables organiza las reglas en diferentes tablas, cada una con un propósito específico:

Tabla Propósito Uso Principal
filter Filtrado de paquetes Permitir/denegar tráfico (tabla por defecto)
nat Traducción de direcciones NAT, redirección de puertos
mangle Modificación de paquetes Alterar headers, QoS
raw Procesamiento temprano Configuraciones especiales, bypass

Cadenas (Chains)

Dentro de cada tabla existen cadenas que representan diferentes puntos en el procesamiento de paquetes:

INPUT → Paquetes dirigidos al sistema local
OUTPUT → Paquetes originados en el sistema local
FORWARD → Paquetes que atraviesan el sistema (routing)
PREROUTING → Procesamiento antes del routing
POSTROUTING → Procesamiento después del routing

Targets y Acciones

Cuando un paquete coincide con una regla, se ejecuta una acción o target:

  • ACCEPT: Permite el paso del paquete
  • DROP: Descarta el paquete silenciosamente
  • REJECT: Rechaza el paquete y envía respuesta de error
  • LOG: Registra información del paquete
  • DNAT/SNAT: Traduce direcciones de destino/origen

3. Instalación y Configuración Inicial

Instalación

En la mayoría de las distribuciones Linux, iptables viene preinstalado. Si no es el caso:

# Ubuntu/Debian sudo apt update && sudo apt install iptables # CentOS/RHEL/Fedora sudo yum install iptables # o sudo dnf install iptables # Arch Linux sudo pacman -S iptables

Verificar Estado

# Verificar si iptables está funcionando sudo iptables -L -v -n # Verificar versión iptables --version
⚠️ Advertencia: Antes de comenzar a configurar iptables, asegúrate de tener acceso físico al servidor o una conexión de consola alternativa. Una configuración incorrecta puede bloquearte el acceso SSH.

4. Sintaxis Básica

La sintaxis general de iptables sigue este patrón:

iptables [-t tabla] [acción] [cadena] [criterios] -j [target]

Parámetros Principales

Parámetro Descripción Ejemplo
-t Especifica la tabla -t filter (por defecto)
-A Agregar regla al final -A INPUT
-I Insertar regla en posición -I INPUT 1
-D Eliminar regla -D INPUT 3
-s Dirección IP origen -s 192.168.1.100
-d Dirección IP destino -d 10.0.0.1
-p Protocolo -p tcp
--dport Puerto destino --dport 80
--sport Puerto origen --sport 1024:65535

5. Reglas Básicas

Ver Reglas Actuales

# Listar todas las reglas con números de línea
sudo iptables -L -n --line-numbers # Ver reglas con estadísticas sudo iptables -L -v -n # Ver reglas de una tabla específica sudo iptables -t nat -L -n

Políticas por Defecto

Las políticas definen qué hacer con paquetes que no coinciden con ninguna regla:

# Establecer política por defecto
sudo iptables -P INPUT DROP
sudo iptables -P FORWARD DROP
sudo iptables -P OUTPUT ACCEPT
# Ver políticas actuales sudo iptables -L | grep policy
💡 Consejo: Siempre permite el tráfico de loopback antes de establecer políticas restrictivas:
sudo iptables -A INPUT -i lo -j ACCEPT

6. Ejemplos Prácticos

Permitir SSH

# Permitir SSH entrante en puerto 22
sudo iptables -A INPUT -p tcp --dport 22 -j ACCEPT

# Permitir SSH solo desde una IP específica
sudo iptables -A INPUT -p tcp -s 192.168.1.100 --dport 22 -j ACCEPT

Configurar Servidor Web

# Permitir HTTP (puerto 80)
sudo iptables -A INPUT -p tcp --dport 80 -j ACCEPT

# Permitir HTTPS (puerto 443)
sudo iptables -A INPUT -p tcp --dport 443 -j ACCEPT

# Permitir solo conexiones establecidas y relacionadas
sudo iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

Bloquear IP Específica

# Bloquear una IP específica
sudo iptables -A INPUT -s 192.168.1.50 -j DROP

# Bloquear un rango de IPs
sudo iptables -A INPUT -s 192.168.1.0/24 -j DROP

Límite de Conexiones

# Limitar conexiones SSH (máximo 3 por minuto)
sudo iptables -A INPUT -p tcp --dport 22 -m limit --limit 3/min -j ACCEPT

# Protección contra ataques de fuerza bruta
sudo iptables -A INPUT -p tcp --dport 22 -m state --state NEW -m recent --set
sudo iptables -A INPUT -p tcp --dport 22 -m state --state NEW -m recent --update --seconds 60 --hitcount 4 -j DROP

7. Configuraciones Avanzadas

NAT (Network Address Translation)

# SNAT - Cambiar IP origen (para salida a Internet)
sudo iptables -t nat -A POSTROUTING -o eth0 -j SNAT --to-source 203.0.113.1

# MASQUERADE - Para IPs dinámicas
sudo iptables -t nat -A POSTROUTING -o ppp0 -j MASQUERADE

# DNAT - Redirección de puertos
sudo iptables -t nat -A PREROUTING -p tcp --dport 80 -j DNAT --to-destination 192.168.1.100:8080

Port Forwarding

# Redirigir puerto 8080 externo al 80 interno
sudo iptables -t nat -A PREROUTING -p tcp --dport 8080 -j DNAT --to-destination 192.168.1.10:80
sudo iptables -t nat -A POSTROUTING -p tcp -d 192.168.1.10 --dport 80 -j SNAT --to-source 192.168.1.1

Logging Avanzado

# Log de paquetes dropped con prefix personalizado
sudo iptables -A INPUT -j LOG --log-prefix "IPTABLES-DROPPED: " --log-level 4

# Log con límite de velocidad para evitar spam
sudo iptables -A INPUT -m limit --limit 2/min -j LOG --log-prefix "IPTABLES-DROPPED: "

Módulos Útiles

Módulo Función Ejemplo de Uso
conntrack Seguimiento de conexiones -m conntrack --ctstate ESTABLISHED
multiport Múltiples puertos -m multiport --dports 80,443,8080
recent Tracking de IPs -m recent --set --name SSH
string Búsqueda de texto -m string --string "malware"
time Restricciones temporales -m time --timestart 09:00 --timestop 17:00

8. Persistencia de Reglas

Por defecto, las reglas de iptables se pierden al reiniciar el sistema. Para hacerlas persistentes:

Ubuntu/Debian

# Instalar iptables-persistent
sudo apt install iptables-persistent
# Guardar reglas actuales
sudo iptables-save > /etc/iptables/rules.v4
sudo ip6tables-save > /etc/iptables/rules.v6

# Restaurar reglas
sudo iptables-restore < /etc/iptables/rules.v4

CentOS/RHEL

# Guardar reglas
sudo service iptables save

# O manualmente
sudo iptables-save > /etc/sysconfig/iptables

# Habilitar servicio
sudo systemctl start iptables

Script de Inicio Personalizado

#!/bin/bash
# /etc/init.d/iptables-rules

# Limpiar reglas existentes
iptables -F
iptables -X
iptables -t nat -F
iptables -t nat -X

# Políticas por defecto
iptables -P INPUT DROP
iptables -P FORWARD DROP
iptables -P OUTPUT ACCEPT

# Permitir loopback
iptables -A INPUT -i lo -j ACCEPT

# Permitir conexiones establecidas
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

# Permitir SSH
iptables -A INPUT -p tcp --dport 22 -j ACCEPT

# Agregar más reglas según necesidades...

9. Mejores Prácticas

🛡️ Seguridad

  • Principio de menor privilegio: Denegar todo por defecto, permitir solo lo necesario
  • Orden de reglas: Las reglas más específicas primero
  • Logging: Registrar tráfico rechazado para análisis
  • Rate limiting: Proteger contra ataques de denegación de servicio

⚠️ Precauciones

  • Siempre mantener una regla de acceso SSH antes de aplicar políticas restrictivas
  • Probar reglas en entorno de desarrollo antes de producción
  • Mantener respaldos de configuraciones funcionales
  • Documentar cambios y propósito de cada regla

Script de Configuración Típica

#!/bin/bash
# Configuración básica de iptables para servidor web

# Limpiar reglas existentes
iptables -F
iptables -X
iptables -t nat -F
iptables -t nat -X

# Políticas por defecto
iptables -P INPUT DROP
iptables -P FORWARD DROP
iptables -P OUTPUT ACCEPT

# Permitir loopback
iptables -I INPUT 1 -i lo -j ACCEPT

# Permitir conexiones establecidas y relacionadas
iptables -A INPUT -m conntrack --ctstate ESTABLISHED,RELATED -j ACCEPT

# SSH (puerto 22) - SIEMPRE PRIMERO
iptables -A INPUT -p tcp --dport 22 -j ACCEPT

# HTTP y HTTPS
iptables -A INPUT -p tcp --dport 80 -j ACCEPT
iptables -A INPUT -p tcp --dport 443 -j ACCEPT

# Ping (ICMP)
iptables -A INPUT -p icmp --icmp-type echo-request -j ACCEPT

# Log paquetes rechazados (opcional)
iptables -A INPUT -m limit --limit 5/min -j LOG --log-prefix "iptables denied: " --log-level 7

echo "Configuración de iptables aplicada correctamente"

Monitoreo y Mantenimiento

# Ver estadísticas de reglas
sudo iptables -L -v -n

# Resetear contadores
sudo iptables -Z

# Monitoreo en tiempo real de logs
sudo tail -f /var/log/syslog | grep iptables

# Ver conexiones activas
sudo netstat -tupln

10. Conclusión

iptables sigue siendo una herramienta fundamental para la seguridad en Linux, ofreciendo un control granular sobre el tráfico de red. Aunque las distribuciones más modernas están migrando hacia nftables, el conocimiento de iptables permanece relevante debido a:

  • Su amplia adopción en sistemas existentes
  • La abundante documentación y ejemplos disponibles
  • Su estabilidad y confiabilidad probada
  • La compatibilidad hacia atrás que mantienen las nuevas herramientas
