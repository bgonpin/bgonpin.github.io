SHA-1 - Algoritmo de Hash

Función hash criptográfica para verificación de integridad

Tipos de Backup Regla 3-2-1 RTO y RPO Software de Backup Verificación de Integridad

¿Qué es SHA-1?

Secure Hash Algorithm 1

SHA-1 es una función hash criptográfica que produce un valor hash de 160 bits (40 caracteres hexadecimales). Fue diseñado por la NSA y publicado por NIST en 1995 como parte de la familia SHA.

Características principales

  • Longitud del hash: 160 bits (40 caracteres hex)
  • Velocidad: Rápido pero más lento que MD5
  • Uso común: Verificación de integridad, firmas digitales
  • Estado actual: Obsoleto para seguridad criptográfica

Cómo funciona SHA-1

Proceso de hash

SHA-1 procesa el mensaje de entrada en bloques de 512 bits, aplicando una serie de operaciones lógicas y matemáticas para producir el hash final.

Pasos principales

  1. Preprocesamiento: Añade padding y longitud del mensaje
  2. Procesamiento en bloques: Divide el mensaje en bloques de 512 bits
  3. Expansión del mensaje: Expande cada bloque a 80 palabras de 32 bits
  4. Compresión: Aplica función de compresión 80 veces por bloque
  5. Resultado: Produce hash de 160 bits

Uso en Verificación de Integridad

Aplicaciones comunes

  • Verificación de descargas: Comprobar integridad de archivos
  • Control de versiones: Git y otros sistemas de control de versiones
  • Firmas digitales: Parte de certificados SSL/TLS (obsoleto)
  • Verificación de backups: Asegurar que los datos no han cambiado

Ejemplo de uso

# Linux
sha1sum archivo.txt

# Windows PowerShell
Get-FileHash -Algorithm SHA1 archivo.txt

# Git
git hash-object archivo.txt

Seguridad y Limitaciones

Vulnerabilidades conocidas

  • Colisiones: Encontradas en 2005 por Xiaoyun Wang
  • Ataques de colisión: SHAttered attack en 2017
  • Longitud insuficiente: 160 bits vs 256+ bits de algoritmos modernos

¿Por qué está obsoleto?

  • No resiste ataques de colisión
  • Reemplazado por SHA-256 en protocolos modernos
  • Prohibido en certificados SSL desde 2017

Alternativas Recomendadas

Algoritmos más seguros

  • SHA-256: Estándar actual para integridad
  • SHA-3: Nuevo estándar NIST
  • BLAKE2: Alternativa moderna y rápida

Migración desde SHA-1

Se recomienda migrar urgentemente a algoritmos más seguros. SHA-1 ya no se considera seguro para aplicaciones criptográficas.

Herramientas que usan SHA-1

Herramientas nativas

  • Linux: sha1sum
  • Windows: CertUtil, PowerShell
  • macOS: shasum

Aplicaciones

  • Sistemas de control de versiones (Git)
  • Herramientas de backup (algunas versiones antiguas)
  • Software de verificación de archivos
